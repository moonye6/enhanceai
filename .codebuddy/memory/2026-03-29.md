# 2026-03-29

## EnhanceAI 项目优化与 OAuth 修复

### 修复的问题

1. **OAuth redirect_uri_mismatch 修复** — `src/app/api/auth/callback/route.ts`
   - 移除硬编码的 `https://enhanceai.pages.dev/api/auth/callback`
   - 改为从 `NEXT_PUBLIC_SITE_URL` 环境变量或 `request.url` 动态推导 redirect_uri
   - 添加 CSRF state 验证（对比 cookie 与 Google 返回的 state）
   - 使用 `crypto.randomUUID()` 代替 `Math.random()` 生成 state
   - 添加 `access_type=online` 和 `prompt=select_account` 参数

2. **创建缺失的 signout 路由** — `src/app/api/auth/signout/route.ts`（新建）
   - 清除 KV 中的 session + 删除 session_id cookie
   - 修复前端 handleLogout 逻辑

3. **移除 next-auth 残留依赖** — `package.json` 中删除 `next-auth`

4. **清理重复配置文件** — 删除 `next.config.js`，将安全 headers 合并到 `next.config.mjs`

5. **客户端限流重构** — `src/lib/rateLimit.ts`
   - 添加 `syncFromServer()` 函数，让服务端 remaining 覆盖客户端缓存
   - 添加注释明确客户端限流仅为 UI 展示

6. **类型安全** — 消除所有 `as any`
   - enhance/route.ts、history/route.ts、session/route.ts、callback/route.ts 全部改用 `Record<string, KVStore>`
   - 添加 `ProRecord`、`EnhanceApiResult`、`HistoryRecord` 等强类型接口

7. **环境变量类型声明** — `src/types/cloudflare.d.ts` 添加 `GOOGLE_CLIENT_ID`、`GOOGLE_CLIENT_SECRET`、`FAL_AI_API_KEY`

8. **多域名自动适配** — 移除 `NEXT_PUBLIC_SITE_URL` 优先逻辑，始终从 `request.url` 动态推导 redirect_uri
   - 测试域名：`enchanceai.pages.dev`
   - 正式域名：`www.enhanceai.online`

9. **回调路径改为 `/api/auth/callback/google`** — 采用多提供商命名风格
   - 删除 `src/app/api/auth/callback/route.ts`
   - 新建 `src/app/api/auth/callback/google/route.ts`
   - 前端 `handleLogin` 跳转路径同步更新

10. **修复 yarn.lock 导致部署失败** — lockfile 中仍残留 next-auth 及其独占依赖（@auth/core, @panva/hkdf, jose@^6, oauth4webapi, preact, preact-render-to-string），Cloudflare Pages 的 `--immutable` 安装模式检测到 lockfile 需要修改而报错。手动清理了所有条目。

### Google Cloud Console 操作提醒
用户需要在 Google Cloud Console → OAuth 2.0 Client → Authorized redirect URIs 中添加**所有**回调地址：
- `http://localhost:3000/api/auth/callback/google`（本地开发）
- `https://enchanceai.pages.dev/api/auth/callback/google`（测试环境）
- `https://www.enhanceai.online/api/auth/callback/google`（正式环境）

11. **页面交互逻辑调整 — 移除登录墙，未登录也展示完整 UI**
    - **`src/app/page.tsx`**：
      - 移除 `if (!user)` 的登录墙，所有用户都能看到完整页面（导航、Hero、上传区、功能展示）
      - 新增 `requireLogin()` 工具函数，在执行实际功能前检查登录状态
      - 「Enhance Image」按钮：未登录点击后跳转 Google 登录，已登录正常执行增强
      - 「Upgrade to Pro」按钮：同样需要登录后才能操作
      - 导航栏：未登录显示 Sign in 按钮，已登录显示 History 链接 + 用户名 + Sign out
      - 新增底部 CTA 区块：未登录用户看到 "Ready to enhance your images?" + 登录按钮
      - 新增 `useSearchParams` 处理 Pricing 页面跳转的 `?upgrade=` 参数
    - **`src/app/history/page.tsx`**：修复登录链接从 `/api/auth/callback` → `/api/auth/callback/google`

12. **修复三大 Bug：无历史记录 / 次数不减少 / 图片未经 AI 处理**
    - **根因**：`FAL_AI_API_KEY` 环境变量未配置时，enhance API 进入 Demo 模式，直接将原图原样返回，且跳过了限流计数和历史记录保存
    - **`src/app/api/enhance/route.ts`**：
      - 重构代码结构，将 Demo / Real 分支统一为设置 `enhancedUrl` 变量，然后共享后续的计数递增 + 历史保存逻辑
      - Demo 模式现在也会正确：递增限流计数、保存历史记录、返回更新后的 remaining
      - 历史记录中 Demo 模式的 enhancedUrl 标记为 `(demo mode)`
    - **`src/app/page.tsx`**：
      - 增强成功后检测 `data.demo` 标记，显示 ⚠️ Demo 提示信息
    - **关键操作**：需要在 Cloudflare Pages 环境变量中配置 `FAL_AI_API_KEY` 才能启用真正的 AI 增强
