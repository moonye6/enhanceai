# 2026-03-30 工作日志

1. **修复 Next.js 15 构建错误：`ssr: false` 不允许在 Server Components 中使用**
   - 错误：`next/dynamic` 的 `ssr: false` 选项在 Next.js 15 的 Server Components 中被禁止
   - 受影响文件：`src/app/page.tsx`、`src/app/history/page.tsx`
   - 修复方案：
     - `page.tsx`（首页）：添加 `'use client'` 指令（该文件无 `export const metadata`）
     - `history/page.tsx`：不能加 `'use client'`（因为有 `export const metadata`），改为：
       1. 新建 `src/components/HistoryLoader.tsx` 客户端包装组件（`'use client'` + `dynamic ssr:false`）
       2. `history/page.tsx` 保持 Server Component，导入 `HistoryLoader` 代替直接 `dynamic`

2. **🚀 性能优化：图片增强流程从"分钟级"优化到"秒级"**
   - **根因**：之前客户端只做了 3MB/2048px 的粗压缩，发给 fal.ai 的图片太大，处理耗时 30-60+ 秒；API 还要再从 fal.ai 下载大图、base64 编码后塞进 JSON response 返回，整个链路极慢
   - **优化方案**：
     1. **客户端压缩**（`compressImageForUpload`）：目标从 3MB/2048px → **500KB/1024px**
        - 1024px 输入 → fal.ai 4x → 4096px 输出，质量够用
        - fal.ai 处理 500KB 图片约 3-8 秒（vs 之前 30-60 秒）
     2. **后端 API 精简**（v7.0.0）：
        - 彻底移除图片下载/base64 中转，直接返回 fal.ai CDN URL
        - API 响应体从几 MB 缩减到几百字节 JSON
   - 文件：`src/components/EnhanceApp.tsx`、`src/app/api/enhance/route.ts`
